> **적용 범위**: `Assets/00.Work/KDS/` 안의 씬·스크립트·SO·아트만.

## 1. 역할

- KDS 담당: **맵 제작 + 스토리 연동 레벨 디자인**
- 작업 경로만 사용: `01.Scene` / `02.Script` / `08.SO` / `05.Asset`(아트)
- 플레이어·좋아요·괴물 **공용 시스템**이 팀원 폴더에 있으면 **참조만** 하고, 복제·이전·수정은 합의 전 금지

## 2. 플레이 레퍼런스

| 레퍼런스 | 가져올 것 | 가져오지 말 것 |
|---|---|---|
| **스마일모 (Smilemo)** | 뒤에서 오는 압박, 앞으로만 달리며 장애물 회피, 스테이지 호흡·수집 배치감 | 튕김 실패 전 맵화, 순수 정밀 챌린지 톤, 사이버 비주얼 전체 복제 |
| **산나비** | 인게임 자막으로 서사 전달 | 시나리오·세계관 차용 |
| **인생게임** | 달리기=인생, 갈림길→엔딩 정산 수치 | 멀티 엔딩 (Follow Me 엔딩은 1개) |

스마일모식 **추격+전진+회피**는 Follow Me의 언팔로워와 맞다.  
다만 추격은 아케이드 기믹이 아니라 **좋아요가 나빠진 대가**로 쓴다.

---

## 3. 핵심 맵 규칙 — 좋아요 상태 ↔ 맵 모드

맵은 한 벌을 깔아 두고, 좋아요 상태에 따라 **장애물을 켜고 끈다.**  
위험해졌다고 **장애물을 갑자기 생성하지 않는다.** (억울한 사망 방지)

### 3-1. 상태

| 상태 | 조건 (의도) | 맵 | 괴물 |
|---|---|---|---|
| **안정** | 좋아요/수집 페이스 양호 | 평탄·일반 러닝. 장애물은 깔려 있어도 **비활성·희미** | 없음 |
| **경고** | 좋아요 악화, 스폰 임박 | 장애물이 **예고**되며 드러남 (실루엣·깜빡임·사운드) | 아직 없음 또는 예고음 |
| **추격** | 언팔로워 출현 | 장애물 **활성** + 전진 강제 회피 | 저항 불가, 회피만 |
| **회복** | 좋아요를 다시 모음 | 장애물 비활성, 일반 맵으로 복귀 | 디스폰 |
| **굴레 (S16만)** | 엔딩 스테이지 | 회복해도 장애물/괴물 **꺼지지 않음** | 디스폰 불가 → 강제 게임오버 = 엔딩 |

### 3-2. 설계 원칙

1. **활성/비활성** — 콜라이더·데미지·위험 플래그만 토글. 타일 자체를 런타임에 끼워 넣지 말 것.
2. **예고 필수** — `안정 → 추격` 직행 금지. 최소 짧은 `경고`로 “맵이 험해진다”를 보여 줄 것.
3. **추격 중 목표** — 장애물을 피하며 앞으로 가고, 좋아요를 모아 `회복`으로 돌아가는 것.
4. **은유** — 숫자가 괜찮으면 도시가 평탄하고, 불안해지면 같은 거리가 장애물로 느껴진다.

### 3-3. Act별 강도

| Act | 스테이지 | 이 시스템 |
|---|---|---|
| 1 처음 | 1~3 | **거의 끔** — 안정 맵만. 위협·장애물 모드 없음 |
| 2 달콤함 | 4~8 | **도입** — 골목·낮은 장애물만. 첫 추격은 그늘 구간 |
| 3 불꽃 | 9~11 | **중강** — 축제 후반·공허 구간에서 추격 비중 증가 |
| 4 터널 | 12~14 | **기본 페이스** — 좁은 맵+장애물 활성 시간이 김. 막차 추격 세트피스 |
| 5 굴레 | 15~16 | 15: Act1 맵 재사용·채도↓·위협 없음 / **16: 회복 불가** |

## 4. 맵 공통 요구 (스토리 문서와 동일)

- 막마다 **갈림길 1~2곳**: 상단=좋아요 밀집 / 하단=일상 오브젝트
- 막마다 **세트피스용 직선 구간 1곳** (대사·음악 동기화, Act4 막차 추격 등)
- **계절 팔레트**: Act1 봄 → 2 여름 → 3 가을 → 4 겨울 → 5 봄(같은 거리, 다른 사람)

## 5. 구현 시 주의 (KDS 스크립트)

- 새 스크립트: `Assets/00.Work/KDS/02.Script/`
- 맵 모드 데이터/튜닝: `Assets/00.Work/KDS/08.SO/`
- 상태 전환은 `[SerializeField]`로 임계값·연출 시간 주입. `FindObjectOfType` 남용 금지
- 좋아요·괴물 API가 팀원 폴더에 있으면 **이벤트/인터페이스로 구독**만 (폴더 침범 금지)
- 코드 수정 후 루트 `AGENTS.md` 5장: Unity MCP `refresh_unity` → `read_console` (에러 0 확인)

- 작업 루트가 `Assets/00.Work/KSY/` 또는 `YHW/` 일 때 → **이 파일 읽기·수정 금지**
